M04L02: CSS: display, overflow [20:22]

życie w pudełku

świat stron internetowych jest płaski (prawie cały)

div - nie ma znaczenia semantycznego, używamy gdy chcemy coś "opakować" (tylko kiedy nie istnieje odpowiedni semantyczny znacznik)

display

display: block - przenosi ostylowany element do nowej lini, element zajmuje całą dostępną szerokość

display: inline - wyświetla elementy obok siebie, elementy są tak szerokie jaki ich zawartość/kontent/treść

display: inline-block - jeżeli elementy są wystarczająco małe, mają wystarczająco wąską zawartość, to zachowują się inlinowo, jeśli są tak szerokie, że nic więcej sie nie zmieści, linia zamienia się w block

wartości width / height określają wysokość i szerokość pudełka. Przy display: inline są ignorowane

wartości max-width / max-height pozwalają na kurczenie się elementów, dzięki czemu na mniejszych ekranach nie pojawia się scroll. Elementy mogą się kurczyć, a jednocześnie nie będą większe niż określona wartość px

min-width / min-height NIE pozwalają na kurczenie się elementów bardziej niż określona wartość px, przez co na mniejszych ekranach może pojawiać się scroll, ale dzięki temu zmieści się kontent

overflow

overflow: hidden; - ucina treść, wszystko co nie zmieści się w pudełku, zostanie ukryte, zamiast wyjechać poza kontener

overflow: auto; - ucina treść, ale pozwala ją przescrollować, scroll jest widoczny cały czas

overflow: clip; - działa podobnie jak hidden, ale jest używane wtedy, gdy niektóre elementy mają wystawać poza body

overflow: scroll; - wszystkie elementy, które nie mieszczą się w pudełku mają scroll, ale jest ładniejszy i pojawia się dopiero po najechaniu kursorem na div

overflow: visible; - kontent, który się nie mieści jest wyświetlany poza pudełkiem, ale to miejsce nie jest rezerwowane, więc

jeżeli pod takim boxem dodamy kolejny div, to ich kontenty się na siebie nałożą i strona będzie nieczytelna



M04L03: CSS: jednostki [10:31]

zero

0 możemy zapisywać bez jednostek

pixele

pixel to jeden punkt na ekranie i nie ma związku z rozdzielczością ekranu. Urządzenia sobie przeliczają

"px" zapisujemy zaraz po wartości, bez spacji, czyli np "200px"

procenty

jest to wartość, która odnosi się do rodzica, zwykle korzystamy z tego przy definicji szerokości width

vw = viewport width

odnosi się do szerokości naszego okna przeglądarki, zapis 50vw spowoduje, że element zajmie połowę szerokości okna.Jeżeli korzystamy z devtools, to wtedy okno jest pomniejszone o narzędzia

vh = viewport height

odnosi się do wysokości naszego okna przeglądarki, zapis 50vh spowoduje, że element zajmie połowę wysokości okna. Jeżeli korzystamy z devtools, to wtedy okno jest pomniejszone o narzędzia



M04L04: CSS: box model [15:47]

BOX MODEL

Wszystkie elementy otoczone są "niewidzialnym" prostokątem

BOX MODEL

padding

padding - zapis pełny / zapis skrótowy (shorthand)

padding - wartości "lustrzane"

padding - wartości wszystkie takie same

border

aby ramka w ogóle się pojawiła trzeba podać dwie wartości: border-width: 10px; border-style: solid

kolor ramki definiujemy za pomocą border-color;

border - zapis pełny / zapis skrótowy (shorthand)

Przy pomocy tego zapisu można stworzyć efekt ramki 3D (8:52 w kursie)

border-radius - odpowiada za zaokrąglenie i nie trzeba mieć zdefiniowanego bordera, żeby skorzystać z tej funkcji

za pomocą border-radius można zrobić kółko

margin

odpowiada za odstęp pudełka od innych elementów

margin - zapisujemy tak samo jak padding

margin: __px auto; - może służyć do wyśrodkowania naszego elementu

M04L06: Position, float [21:04]

position

position: static - przeglądarka czyta wszystko od góry do dołu i wyświetla tak jak napisaliśmy

position: relative - pozwala nam stworzyć przestrzeń, względem której będziemy umieszczać elementy. Samo dodanie tej właściwości nic nie zmienia, trzeba dodać jeszcze element o właściwości position: absolute

position: absolute - właściwość dodajemy do drugiego diva, dzięki czemu możemy sterować położeniem jedengo diva w względem drugiego, który ma właściwość position: relative

top, right, bottom, left - wartości dodajemy do elementu z właściwością position: absolute

z-index - odpowiada za kolejność warstw, im wyższa wartość z-index, tym bardziej na wierzchu

w przeglądarce Edge jest możliwość podejrzenia strony w 3D, z uwzględnieniem warstw

position: fixed - odpowiada za przyklejenie elementu do naszego view port (okna przeglądarki)

position: sticky - pozwala przykleić element w środku kontenera, działa podobnie jak fixed, ale dla konkretnego diva


https://caniuse.com/#feat=css-sticky

float - właściwość pozwala, żeby element był opływany przez tekst

FLOAT NIE SŁUŻY DO BUDOWANIA LAYOUTÓW

clear - własność clear czyści własność float

JS: typy danych i immutability

Zmienna to takie połączenie z naszym wszechświatem wartości

Zmienna to nie wartość!

Zmienne w JS: LET vs CONST vs VAR

VAR już raczej nie używamy

Can I use...?
https://caniuse.com/#feat=es6

Zmienne w JS

// const - zmienne mające stałe połączenie z jedną wartością

const name = 'Maciek';
const age = 35;

// let - zmienne, których połączenie możemy przepinać, ale definiujemy
// tylko raz!

let isHappy = false;
isHappy = true;
isHappy = 'tak';

JS jest językiem nietypowanym. Zmienne w JS są jak młody pelikan: "łykną" wszystko

Typy proste w JS (primitives)

String - tekst, otaczamy ' lub " (zalecam ')
Number - liczba
Bigint - duża liczba
boolean - true lub false
null - wartość celowo niezdefiniowana
undefined - brak definicji
*symbol - tworzy "unikalne opakowanie" - z tego nie korzystamy

TYPEOF - Jakiego typu jest zmienna?

IMMUTABILITY - Nie możemy zmieniać wartości primitive values w JS. To co zmieniamy to połączenie między nimi, a zmiennymi.
