M05L06: JavaScript: funkcje [16:10]

JavaScript: Funkcje

fun('Hi!') Hello world

console.log(); jest funkcją

Czym jest funkcja?

Czym jest funkcja?

f(x) = x / argument i wynik
f - nazwa funkcji
(x) - argument
x - wynik

Czym jest funkcja?

f(x) = x*7 / argument i wynik

Czym jest funkcja?

  • Na wejściu dostaje dane - argumenty
  • Dla podanego argumentu zwraca jakąś wartość
  • określony wzór

Funkcja w JS

function calculate(myNumber) {
return myNumber*7;
}

Funkcja w JS

Deklaracja • Parametr • Słowo kluczowe • Wynik

słowo kluczowe return

Funkcja w JS

function calculate(myNumber) {
myNumber = myNumber + 3;
console.log(myNumber);
return myNumber*7;
}

Wywołanie funkcji w JS

function calculate(myNumber) {
myNumber = myNumber + 3;
console.log(myNumber);
return myNumber*7;
}

calculate(2); // 35
calculate(1); // 28

Funkcja w JS

  • Deklarujemy nazwę funkcji (nie zawsze)
  • W środku dokonujemy obliczeń / wykonujemy zadania
  • Funkcja przyjmuje argumenty (ale nie musi)
  • Funkcja zwraca zawsze coś (nawet jeśli nie mamy return - wtedy funkcja zwraca undefined)

Funkcje po co?

  • Deklarujemy kod, który się powtarza (DRY - don't repeat yourself)
  • Zwiększamy czytelność kodu
  • Łatwiejsze szukanie błędów
  • Odpowiedzialność za wykonywanie zadania w jednym miejscu

(Fat) arrow functions =>

(Fat) arrow function

const calculate = (myNumber) => {
myNumber = myNumber + 3;
console.log(myNumber);
return myNumber*7;
}
calculate(2); // 35

(Fat) arrow function

const calculate = myNumber => (myNumber+3)*7;
calculate(2); // 35

(Fat) arrow function

  • Skraca zapis deklaracji funkcji
  • Zwiększa czytelność (dyskusyjne)
  • Nie tworzy nowego kontekstu this, nie ma dostępu do arguments i super
  • Nie jest zamiennikiem "tradycyjnej" funkcji (ale w większości przypadków nie zobaczysz różnicy)
